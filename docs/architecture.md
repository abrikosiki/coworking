# Архитектура приложения

## Context
Это нетворк-сервис для коворкинга, база данных Supabase. Существуют две роли (`admin` и `member`), логика разделена на авторизацию и получение публичных профилей. Код реализует страницу входа, ленту резидентов и профиль пользователя.

## Сущности базы данных (Supabase)

Проект использует связку встроенной аутентификации Supabase и кастомной таблицы профилей:
- `auth.users` — системная таблица Supabase для хранения пользователей (email, пароль/magic link).
- `public.profiles` — кастомная таблица для данных пользователя (содержит обязательную колонку `role` со значениями `admin` или `member`, а также поля для навыков и контактов).

## Структура приложения (Next.js App Router)

Приложение разделено на несколько основных функциональных зон:
- **Страница логина/регистрации:** `src/app/register/page.tsx` и `src/app/login/page.tsx` (или корень) для входа и регистрации.
- **Лента резидентов коворкинга:** Основная страница или защищенный роут, где юзеры могут видеть список специалистов, их навыки и контакты.
- **Профиль пользователя:** Личный кабинет для редактирования своих навыков, контактов и просмотра чужого профиля.
